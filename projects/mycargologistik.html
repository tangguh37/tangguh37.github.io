Problem

A freight forwarding company (MyKargo Logistik Yogyakarta) needed a monthly logistics performance dashboard. The data was locked in Excel reports with sensitive customer information — requiring anonymization before building a reusable analytics system.

This project transforms Excel reports → anonymized mock data → PostgreSQL → Metabase dashboards for real-time KPI tracking.

Tech Stack

  • PostgreSQL — Relational database for structured storage
  • Metabase — Open-source BI tool for dashboarding
  • Python 3 (stdlib) — Anonymization script with zero dependencies
  • Excel — Source data cleaning and preparation
  • SQL — Table creation, data loading, and analytical queries

What I Built

Designed the anonymization algorithm in Python: 180 unique customers with 2 "whales" generating ~80% revenue, 9 salespeople across 3 performance tiers, realistic job-type mix (~40% Export, ~25% Import, ~35% Other), monthly seasonality patterns, and ~24% target margin. Built the PostgreSQL schema with a single monthly_reports table (11 columns). Created 3 Metabase dashboards: Overall Performance (revenue/cost trends), This Month Snapshot (real-time KPIs), and Marketing Performance (MKT team tracking).

Dashboard Preview

Overall Performance dashboard: revenue and cost trends, monthly KPIs, and margin analysis
This Month Snapshot: real-time monthly revenue, job types, and sales team performance
Marketing Performance: MKT team tracking, customer acquisition, and new business metrics

Learnings

This project taught me how to handle sensitive data responsibly — anonymization is a critical skill for real-world data work. Building realistic synthetic data requires understanding the business domain deeply. Metabase is surprisingly powerful for its simplicity; dashboards can be built in minutes once the schema is right.

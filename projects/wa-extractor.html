WhatsApp Timestamp Extractor

PythonFastAPIExtend AISupabaseGoogle SheetsTesseract OCR
Completed

Problem

Photos shared in WhatsApp groups often contain visible timestamps (e.g., delivery receipts, event photos), but extracting them manually is tedious. This tool automates the process: OCR-read timestamps from images and cross-reference them with the WhatsApp chat export to identify senders.

Architecture

┌───────────────────┐ POST /webhook ┌────────────────────────────┐ │ wa-bridge │──────────────────────▶│ FastAPI Worker (:8080) │ │ (whatsapp-web.js) │ │ │ │ │ │ save_media() │ │ Captures photos │ │ ↓ │ │ from groups │ │ Extend AI OCR (primary) │ └───────────────────┘ │ │ │ ↓ │ │ ┌────┴────┐ │ │ ↓ ↓ │ │ Supabase Google Sheets │ │ (messages (append row) │ │ + OCR) │ └────────────────────────────┘

Two modes available: Live (wa-bridge → webhook → real-time OCR) and Batch (WhatsApp export ZIP → OCR → Excel report). Tesseract OCR is available as a fallback engine in batch mode.

Features

  • Real-time OCR via Extend AI API with high accuracy on timestamp text
  • WhatsApp Web integration — captures photos automatically from groups
  • Dual storage: Supabase for queryable records, Google Sheets for live reporting
  • Batch mode for processing WhatsApp export ZIPs
  • Date/time pattern recognition from raw OCR output
  • Excel report generation with sender, photo, and extracted timestamps

Tech Stack

  • Python 3 — Core programming language
  • FastAPI — Web server for live webhook pipeline
  • Extend AI OCR API — Primary OCR engine for real-time timestamp extraction
  • Tesseract (tesserocr) — Fallback OCR engine for batch mode
  • Supabase — Stores scraped messages and OCR results
  • Google Sheets API — Appends OCR results as rows for live reporting
  • Pillow — Image preprocessing (grayscale, contrast, threshold)
  • OpenPyXL — Excel report generation (batch mode)
  • whatsapp-web.js — WhatsApp Web client for the live bridge

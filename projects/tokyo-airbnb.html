Problem

How do you build a production-grade analytics pipeline for Tokyo Airbnb data — from raw CSV ingestion to a live interactive dashboard — using the modern data stack with dbt, DuckDB, and Airflow?

This project ingests Inside Airbnb data (Tokyo, September 2025), transforms it through a dbt star-schema model, orchestrates everything with Apache Airflow, and visualizes insights via a Streamlit dashboard.

Tech Stack

  • DuckDB — Embedded OLAP database for storage and querying
  • dbt-core + dbt-duckdb — Transformation with medallion architecture
  • Apache Airflow — DAG orchestration for the full pipeline
  • Streamlit + Plotly — Interactive dashboard with visualizations
  • Python — Data download and ingestion scripts
  • Source — Inside Airbnb (Tokyo, September 2025)

What I Built

Built a fully orchestrated analytics pipeline: Python script to download and ingest Airbnb CSVs into DuckDB raw tables, then a dbt project with staging views, intermediate enrichment, and mart tables following a star schema. Implemented generic and singular tests, source freshness checks, incremental models for fact tables, and dbt docs with exposure linking to the Streamlit dashboard. Airflow orchestrates the entire DAG from download to dashboard refresh.

Dashboard Preview

Overview dashboard: pricing by neighbourhood, room type distribution, and key market metrics
Host analysis: Superhost vs Regular host comparison, rating tiers, and booking performance
Market insights: seasonality trends, neighbourhood arbitrage, and strategic recommendations

Learnings

This project solidified my understanding of the medallion architecture (bronze → silver → gold) in practice. Key takeaways: DuckDB + dbt is a powerful lightweight combination for local analytics, Airflow DAGs need careful dependency management, and Streamlit + Plotly enables rapid dashboard prototyping without a heavy frontend framework.
